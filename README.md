# MA Apparel Pro — Custom Sports Uniform Website

A professional, modern B2B website for a **custom sports uniform manufacturer**,
built with **Next.js 14 (App Router)**, **TypeScript**, **Tailwind CSS**,
**Framer Motion** and shadcn-style UI. No cart/checkout — customers contact via
**WhatsApp** or **Email** for free design mock-ups and a bulk quote.

## Quick start

```bash
npm install
npm run dev      # http://localhost:3000
npm run build    # production build
npm run start    # serve production build
```

## Edit your business details (one place)

All branding/contact placeholders live in [`lib/config.ts`](lib/config.ts):

```ts

  COMPANY_NAME: "MA Apparel Pro",
  DOMAIN: "maapparelpro.com",
  LOGO: "/details/logo.png",
  WHATSAPP_NUMBER: "+92XXXXXXXXXX",   // <-- add your real WhatsApp number
  BUSINESS_EMAIL: "sales@maapparelpro.com",
  MIN_ORDER: 10,
  FACEBOOK_URL: "#",                  // <-- add your social links

```

> ⚠️ Set a real `WHATSAPP_NUMBER` (international format) so WhatsApp links work.

## Catalog & images (auto-generated)

The catalog in [`lib/data.ts`](lib/data.ts) is **generated by scanning the
`public/` folder** — so it always matches your real product images.

```
public/
  SPORTS CETAGORY/      # category cover images (one per sport)
  <sport> uniform/
    products/           # full uniforms      -> "Full Uniforms"
    shirts/             # jerseys & shirts    -> "Jerseys & Shirts"
    shorts&pent(s)/     # shorts & pants      -> "Shorts & Pants"
    hoodie/hodies/      # hoodies & outerwear -> "Hoodies & Outerwear"
    accessries/         # accessories         -> "Accessories"
  details/              # logo + process/feature images used on the homepage
```

Files named `back.webp`, `side.webp`, `back (1).webp`, etc. are treated as
alternate-view shots and skipped; every other image becomes a product, with its
name derived from the filename.

**After adding/renaming images, regenerate the catalog:**

```bash
node scripts/gen-data.mjs
```

It prints a per-category product count. Category metadata (display name, icon,
gradient, order) is defined at the top of
[`scripts/gen-data.mjs`](scripts/gen-data.mjs).

## Structure

| Path | Purpose |
| --- | --- |
| `app/page.tsx` | Homepage (hero, stats, featured, fabric features, why-us, how-it-works, testimonials, CTA) |
| `app/categories/page.tsx` | All 12 categories with live search |
| `app/categories/[slug]/page.tsx` | Per-category product grid grouped by type |
| `app/about/page.tsx` | Company story, mission, markets, quality, values |
| `app/contact/page.tsx` | WhatsApp/Email CTAs + mailto inquiry form |
| `components/*` | Navbar, Footer, WhatsAppButton, ProductCard, CategoryCard, QuoteModal, SpotlightCard, Flag, etc. |

Design: deep navy `#0A1628` + electric green `#00D563`, Poppins font, fully
responsive, animated aurora/grid backgrounds, scroll reveals, spotlight-hover
cards, count-up stats, and a floating pulsing WhatsApp button on every page.
Real SVG flags (via `flag-icons`) render correctly on Windows.
